??? Rider Flow

Check orders assigned to them.

Confirm they'll deliver ? status Assigned (2) ? Accepted (3).

PUT /api/orders/{orderId}/pickup Rider Only

Mark package collected ? status Accepted (3) ? PickedUp (4).

??? Key Design Notes

  • Security: Rider / Manager IDs come from the JWT token (via CurrentUserService), not request bodies � this prevents ID spoofing.
  • Role enforcement: [Authorize(Policy = "ManagerOnly")] and "RiderOnly" guard each endpoint.
  • Order lifecycle: Strictly sequential � each status transition (Pending ? Assigned ? Accepted ? PickedUp ? Delivered) is validated inside its handler.
  • Token rotation: Each refresh issues a brand-new refresh token for added security.
  • Real-time: SignalR pushes live rider locations to managers instantly � no polling required.
